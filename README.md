# 0号隔离区（The Quarantine Zero）— FPS Demo 项目 README

## 1. 项目概述

**《0号隔离区》**是我在**腾讯游戏客户端菁英班**期间完成的大作业项目：一款支持**单人 / 局域网多人联机**的**第一人称射击（FPS）Demo**。  
这是我**第一次真正从零开始做游戏**。从搭建场景、拼出玩法循环、处理怪物行为、到联机同步与音效动画，我在不断“做出来—崩掉—修好—再做出来”的过程中，第一次感受到游戏开发的完整链路，也学到了非常多。

虽然最终成果仍然存在不少不完善之处（例如部分流程细节、体验打磨和一些 bug），但这次项目对我来说意义非常大：它为我未来的职业规划打开了一扇新的门，让我更明确自己想要继续在游戏开发方向深入探索。

在此也特别感谢**腾讯的各位老师**在课程期间给予的指导与建议，让我能把这个“第一次”的作品真正落地成一个可玩的 Demo。  

---

## 2. 游戏简介

在一片被生化灾害摧毁的废墟中，**变异怪物**占据了建筑群的每一处角落。玩家将扮演一名持枪的英雄，深入危机四伏的废墟建筑，目标只有一个：

> **找到生化炸弹，并在倒计时结束前成功拆除，保卫世界和平。**

在游玩过程中，玩家需要时刻管理自己的**血量（HP）**与**弹药（Ammo）**：  
怪物会主动靠近并攻击玩家，造成血量下降；弹药不足则会显著增加生存压力。  

---

## 3. 核心玩法循环（Gameplay Loop）

1. 进入关卡（单人 / 联机房间）
2. 在废墟建筑中探索、移动、搜索目标区域
3. 遭遇怪物 → 射击战斗 → 控制血量与弹药
4. 找到炸弹 → 进行拆除
5. 在规定时间内完成拆除即胜利，否则失败

---

## 4. 游戏模式与关卡

- **关卡数量：1 个**
- **模式：**
  - 单人模式：独自完成探索与拆弹
  - 局域网多人模式：在同一局域网内组队协作完成任务

---

## 5. 胜负逻辑（Win / Lose Conditions）

### ✅ 胜利条件
- 在**计时器归零前**找到**生化炸弹**并**成功拆除** → 判定胜利

### ❌ 失败条件
满足任意条件即失败：
1. **玩家血量归零**（被怪物击杀） → 游戏结束  
2. **计时器归零且未拆除炸弹** → 游戏结束  

---

## 6. 主要功能说明

### 6.1 第一人称控制与射击
- 第一人称视角移动、转向、开火
- 子弹数量管理：射击会消耗弹药；弹药不足会限制持续输出能力
- 基础 UI 展示：血量、弹药、倒计时（或以 HUD 形式呈现）

### 6.2 怪物 AI（基础行为）
- 怪物会在场景中**主动靠近玩家**
- 接近到一定范围后触发攻击行为
- 攻击命中后扣除玩家一定血量
- 当玩家血量为 0 时触发死亡与结算

> 说明：本项目以“能跑通流程”为优先，AI 为基础追击 + 近战攻击逻辑。

### 6.3 计时器与任务目标
- 进入游戏即开始计时
- 计时归零时检查任务是否完成
- 未拆除炸弹则判失败
- 找到目标后可触发拆弹流程（交互或触发器方式）

### 6.4 局域网联机（LAN Multiplayer）
支持同一局域网内多人协同作战，包含两种加入方式：
- **搜索并加入房间**（LAN Session 浏览/发现）
- **通过输入 IP 直接加入指定房间**

## 7. 使用到的技术与实现方式（基础 UE 蓝图为主）

> 备注：因为这是我第一次做游戏，本项目**以最基础、可落地的 UE 蓝图实现为主**，没有使用复杂/高级的系统或自定义底层框架。

### 7.1 UE 蓝图系统（核心开发手段）
- 角色控制、输入绑定、射击逻辑、弹药扣减
- UI（HUD）数据绑定：血量/弹药/倒计时显示更新
- 关卡触发器（Trigger）与交互逻辑（例如进入范围、按键交互触发拆弹）

### 7.2 基础怪物行为（蓝图 AI 逻辑）
- 使用蓝图实现怪物的：
  - 目标追踪（持续朝玩家移动）
  - 距离判断（进入攻击范围触发攻击）
  - 伤害结算（对玩家扣血）
- 玩家受击后：
  - 扣除血量
  - 更新 HUD
  - 血量归零触发失败流程

### 7.3 动画与音效触发（蓝图事件驱动）
- 射击动画 / 开火音效：
  - 在“开火事件”触发时播放
- 怪物攻击动画 / 受击反馈：
  - 在攻击或命中事件发生时播放对应动画/音效
- 拆弹交互反馈：
  - 进入交互状态或拆除成功时播放提示音/效果

### 7.4 联机同步（局域网，多人协作）
为了保证多人联机时体验一致，本项目在“**关键玩法事件**”上进行了基础同步（蓝图实现）：

- **房间创建 / 搜索 / 加入：**
  - 使用 UE 的 **LAN 会话（Session）**思路进行房间发现与加入
  - 支持输入 IP 的直连方式进入房间

- **关键状态同步（基础 Replication 思路）：**
  - 玩家关键数据：如血量/死亡状态等（作为需要一致的状态进行同步）
  - 关键行为事件：如开火、命中、受伤、拆弹成功等（作为需要广播的事件进行同步）
  - UI 展示数据由同步后的状态驱动刷新（血量、倒计时等）

- **动画与音效同步（事件广播方式）：**
  - 本项目采用“**在关键事件发生时进行广播**”的方式，确保其他玩家也能看到/听到一致的反馈
  - 例如：
    - 玩家开火 → 触发开火事件 → 同步给其他客户端播放对应动画/音效
    - 怪物攻击命中 → 结算伤害并同步状态 → 播放受击相关反馈

> 总体原则：以 UE 最基础的“状态同步 + 事件触发”方式确保多人协作下的核心体验一致。

## 8. 已知不足（诚实说明）
由于是第一次完整制作游戏 Demo，目前仍存在一些不足：
- 细节打磨有限：例如节奏、反馈、引导等还可更完善
- 可能存在未覆盖到的 bug：包括联机边界情况、流程异常等
- 内容量较少：当前为单关卡 Demo，后续如扩展会加入更多区域与玩法变化
